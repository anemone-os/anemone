# 活动登记册

活动登记册记录当前仍然开放的问题与限制，但不承担长期历史归档，也不替代 GitHub issue / PR 的任务管理职责。

它的目标是提供一份可以快速扫描的当前状态视图，让开发者和 agent 先回答三件事：

- 现在还有哪些开放问题；
- 当前有哪些被接受但尚未消除的限制；
- 哪些事项会改变当前的诊断、验证或兼容性判断。

登记册条目使用文段形式维护，而不是表格。这样在问题开始变复杂时，不需要为了适配列结构而牺牲描述精度。

如果某个事项已经有 GitHub issue / PR，登记册只保留短摘要与链接，不复制完整讨论。

## 子页面

- [开放问题](./register/open-issues.md)：记录本来应该工作、但当前未正确工作的事项。
- [当前限制](./register/current-limitations.md)：记录当前已接受、但尚未补齐的能力缺口或阶段性语义缩减。

## 维护规则

- 问题修复后，从登记册中移除，并把长期历史保留在开发日志中。
- 限制被取消、扩大或重定义后，更新对应条目。
- 如果某一页当前没有条目，明确写出“当前无项”，避免让读者误判为文档不完整。
- 条目格式以模板页中的“问题或限制条目”为准。
