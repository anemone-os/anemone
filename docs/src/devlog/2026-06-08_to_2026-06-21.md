## 2026-06-08 - 空 iomux 超时睡眠修复

**Date:** 2026-06-08
**Authors:** doruche, Codex
**Area:** fs / iomux / ppoll / pselect6 / scheduler

**Summary:** `ppoll` / `pselect6` 的 shared iomux wait helper 新增空集合 / 无 source timeout sleep 路径：`timeout == 0` 仍立即返回，正 timeout 或 NULL timeout 会进入 wait-core interruptible wait，避免 `select(0, ..., 1us)` 在 iozone-glibc 启动同步中退化为纯 busy loop。

**Validation:** `just fmt kernel` 通过；sandboxed `just build` 在 vendored `lwext4_rust` C 编译处命中已知 `Bad system call`；unsandboxed `just build` 通过，仅保留既有 `sync/mono.rs` unused-import warning。未在本轮 agent 侧重跑 iozone QEMU profile。
