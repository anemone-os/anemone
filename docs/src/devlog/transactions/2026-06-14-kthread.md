# 2026-06-14 - KThread

**Status:** Complete for retrospective documentation

## Scope

本事务记录 commit `2b0e3900279895c3d8eb604e463249a02c3bddc9` 中落地的 kthread 基础设施：

- boot-time `kthreadd` 创建代理；
- `KThreadBuilder` 和 typed entry shim；
- ordinary kthread stop / park / wake / exited 生命周期；
- task-owned `Arc<KThread>` 与 weak external handle；
- `KThreadService`、merge slot pending backend 和 FIFO pending backend；
- `kernel_exit()` 对 ordinary kthread finish path 的断言；
- 首个 ordinary kthread consumer inode shrinker 的 boot 接入。

非目标：

- 不提供完整 kthread 兼容 API。
- 不实现通用 workqueue、freezer、cgroup 或 CPU hotplug 语义。
- 不引入用户可见 ABI。

## Invariants

- ordinary kthread 创建经由 `kthreadd`，`kthreadd` 不拥有 ordinary kthread lifecycle state。
- `Task` 强持有 `KThread`，`KThread` weak 引用 `Task`，外部 `KThreadRef` 是 weak handle。
- `KThreadStart<A>` 只由创建失败路径或匹配 typed shim 回收一次。
- `KThreadControl` 是 stop / park / exited 状态的单一真相源。
- entry 返回后先 `finish_returned_entry()`，再 `kernel_exit()`。
- `KThreadService` pending backend 是 work truth，wake event 只提示 worker 重新检查。

## Phase Log

### 2026-06-14 - basic shrinker and kthread

**Phase:** implementation / commit import

**Change:** 新增 `task::kthread` 模块，包含创建代理、typed entry shim、ordinary kthread lifecycle 和 service worker 抽象。

**Change:** `Task` 增加 `kthread: SpinLock<Option<Arc<KThread>>>`，作为 ordinary kthread state 的强 owner；`KThread` 只保存 weak task pointer，外部 `KThreadRef` 也是 weak handle。

**Change:** `kernel_exit()` 增加 `assert_kthread_exit_ready()`，要求 ordinary kthread 在进入 task exit 前已经通过 kthread finish path 写入 `Exited`。

**Change:** boot path 在 BSP 初始化中创建 `kthreadd`，并在所有 CPU 完成本地初始化后启动首个 ordinary kthread consumer。

**Review:** 文档追补时未发现需要单独 tracking 的 confirmed design issue。关键协议已折入 RFC index、invariants 和 implementation 文本。
