## 2026-07-08 - Sched Split Zombie Tail Feedback

**Date:** 2026-07-08
**Authors:** doruche, Codex
**Area:** scheduler / task exit / kernel preempt

**Summary:** `sched-wait-preempt-arming` 收口后的单核 `kernel_preempt` 试运行暴露
`schedule_preempt cannot preempt zombie current task` panic。复核确认旧 exit 尾部先发布
`TaskSchedState::Zombie`、再进入 no-return schedule，中间可被 trap-tail preempt 观察到 zombie
current。本次反馈已回写 RFC：最终 `Runnable -> Zombie` 发布归 scheduler core 的 noirq
`schedule_zombie_never_return()` 事务；`schedule_preempt()` 仍不接受 zombie current。

**Validation:** `just fmt kernel`、`just build`、目标源码 `git diff --check`、全仓
`git diff --check` 和 `mdbook build docs` 通过。agent 未运行 QEMU / LTP / iozone runtime
复现。

**Related:** [Sched Wait Preempt Arming 事务日志](./transactions/2026-07-06-sched-wait-preempt-arming.md), [RFC-20260618-sched-wait-preempt-arming](../rfcs/sched-wait-preempt-arming/index.md)

**Date:** 2026-07-12

**Area:** scheduler / fairness / runtime accounting / performance

**Summary:** 用户运行证据证明 Stage 3 的 monotonic minimum-`vruntime` eligibility

R3b coordinate rebase。default EEVDF 目标不回退，也不接受 forced handoff 或 penalty tuning 窄修。
