# ANE-CHG-20260614-timerfd

**Type:** Small Feature / Syscall / Anonymous FD
**Status:** Completed
**Date:** 2026-06-14
**Authors:** doruche, Codex
**Area:** VFS / FileOps / timer / iomux / syscall ABI / LTP

## Problem

rv64 / la64 的 asm-generic syscall table 之前没有 `timerfd_create`、
`timerfd_settime` 和 `timerfd_gettime`。LTP timerfd 用例会直接覆盖 syscall number、
fd flags、nonblocking read、periodic expiration、`gettime` snapshot、bad fd / bad
pointer errno，以及 `TFD_TIMER_CANCEL_ON_SET` flag 边界，因此单纯补 syscall stub 不够。

timerfd 适合作为小迭代处理：匿名 `STREAM` fd、opened-description status snapshot、
latch-style poll registration 和 IRQ timer event 已经存在。主要风险在于 timerfd readiness
会从 timer interrupt callback 异步变化，不能把状态散落到 VFS cursor 或 fd status 镜像里。

## Scope

本轮实现基础 timerfd fd object：

- rv64 / la64 新增 `SYS_TIMERFD_CREATE = 85`、`SYS_TIMERFD_SETTIME = 86`、
  `SYS_TIMERFD_GETTIME = 87`。
- 新增 Linux `ITimerSpec`、`TFD_*` 常量和 `CLOCK_BOOTTIME = 7`。
- `CLOCK_REALTIME`、`CLOCK_MONOTONIC` 和 stage-1 `CLOCK_BOOTTIME` 可作为 timerfd clock。
- 新增 timerfd anonymous `STREAM` file 后端，覆盖 `read`、blocking / nonblocking、
  readable poll/select、one-shot、periodic、relative 和 absolute timer。
- `timerfd_create()` 校验 clock/flags，并映射 `TFD_CLOEXEC`、`TFD_NONBLOCK` 到 fd flags
  与 opened-description status。
- `timerfd_settime()` 支持 arm、disarm、old snapshot、generation-based stale callback
  filtering 和 `TFD_TIMER_CANCEL_ON_SET` accepted no-op。
- `timerfd_gettime()` 返回 remaining time 和 interval snapshot。
- LTP fixture 声明 `CONFIG_TIMERFD=y`，并新增 stage-1 timerfd group。

本轮不实现通用 cancellable timer core、threaded timer event、alarm clock、
真实 suspend/resume boottime accounting、time namespace、clock-set cancellation 后的
`read() -> ECANCELED`、timerfd fdinfo、`TFD_IOC_SET_TICKS` ioctl、time32/time64 compat
alias 410/411，或 timerfd 直接 userspace I/O 精确 copy 语义。`timerfd04` 依赖 time
namespace，不是本轮 expected pass。

## Solution

timerfd 作为 anonymous stream file 实现。VFS inode/path 只提供 fd 身份；expiration
counter、schedule、blocking wait、poll readiness 和 logical cancellation 都由
`fs::timerfd` 私有 state owner 负责。

`TimerFdSchedule` 是 arm/disarm、下一次到期点和 interval 的唯一真相源。每次
`timerfd_settime()` 推进 generation；IRQ callback 只捕获 `Weak<TimerFdCore>` 和
generation，旧 callback 如果 generation 不匹配或 fd core 已释放，就直接丢弃。

timerfd state 使用 `NoIrqSpinLock`，因为到期 callback 运行在 timer interrupt context。
callback 在锁内只做 generation 校验、O(1) expiration accounting、schedule 更新和 trigger
detach；read/poll waiters 在锁外唤醒。注册路径会把 stale trigger 搬到固定容量 batch，
离开 no-IRQ lock 后再 drop，避免在 IRQ-disabled 临界区释放 wait-core token state。

`CLOCK_BOOTTIME` 当前是 monotonic-backed alias，只服务本轮 clockid 兼容。后续引入
suspend accounting、time namespace 或 threaded timer event 后，需要重新审查 absolute
timer、clock-set cancellation 和 periodic rearm 的 owner boundary。

## Change

- `anemone-abi` 为 rv64 / la64 新增 timerfd syscall numbers，并新增 `ITimerSpec`、
  `CLOCK_BOOTTIME` 与 `TFD_*` UAPI 常量。
- `time::clock` 增加 stage-1 `CLOCK_BOOTTIME` monotonic-backed alias，并在代码注释中说明
  suspend/time-namespace 限制。
- 新增 `fs::timerfd` 后端模块：anonymous inode 构造、`FileMode::STREAM` opened file、
  timerfd state、read/poll FileOps、generation stale filtering、periodic accounting 和
  no-IRQ trigger detach。
- 新增 `fs::api::timerfd/` 目录，按一个 syscall 一个文件组织
  `create.rs`、`settime.rs`、`gettime.rs`，`mod.rs` 只承载共享 flag parsing。
- LTP fixture 新增 `CONFIG_TIMERFD=y`，user-test LTP groups 新增 `timerfd` group。

## Tracking Issues

### CHG-001 - no-IRQ stale trigger drop

**Status:** Neutralized
**Severity:** Keter

**Issue:** timerfd read/poll trigger pruning 如果在 `NoIrqSpinLock` 内用 `retain()` drop
stale `LatchTrigger`，可能在 IRQ-disabled 临界区释放 wait-core token state。

**Resolution:** 注册路径改为把 stale entries 搬入固定容量 `TimerFdTriggerBatch`，离开
state lock 后再 drop。到期路径不再 prune，只 detach 全部 read/poll waiters，并在锁外
trigger。

### CHG-002 - timerfd precise zero-length I/O

**Status:** Deferred
**Severity:** Euclid

**Issue:** Linux timerfd `write()` 返回 `EINVAL`，且 `read()` 对 `count < 8` 返回
`EINVAL`。当前 Anemone generic I/O helper 在 `count == 0` 时先返回 `0`，timerfd 后端
看不到该请求。

**Resolution:** 这是已知 stream-fd I/O 精度缺口，本轮不为 timerfd 单点扩展 direct
userspace I/O hook。后续应与 eventfd 的 zero-length、`readv/writev` 聚合和 scalar
尾部坏地址语义一起统一收敛。

### CHG-003 - threaded timer event

**Status:** Deferred
**Severity:** Safe

**Issue:** 当前 periodic timerfd 仍通过现有 IRQ timer event 做一次 O(1) rearm。它不引入
timerfd 私有分配或循环补排，但仍不是长期的 threaded/cancellable timer infrastructure。

**Resolution:** 本轮保留 stage-1 bridge。引入 threaded timer event 或 cancellable timer
core 后，再迁移 timerfd rearm、clock-set cancellation 和 stale event cleanup。

## Risk / Follow-up

- `timerfd01`、`timerfd02`、`timerfd_create01`、`timerfd_gettime01`、
  `timerfd_settime01` 和 `timerfd_settime02` 是第一轮 LTP 目标；runtime 结果需由后续
  QEMU / LTP profile 验证。
- `TFD_TIMER_CANCEL_ON_SET` 当前只作为 documented no-op 接受，不产生 `ECANCELED`。
- `CLOCK_BOOTTIME` 当前不包含 suspend time，也没有 time namespace offset。
- 当前没有声明 `CONFIG_TIME_NS=y`，避免把 `timerfd04` 误归入基础 timerfd 能力。
