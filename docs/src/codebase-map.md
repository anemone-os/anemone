# 代码库地图

这一页刻意保持浅。

目标不是解释每一个模块，而是让人或 agent 在真正读代码前，先快速找到 owning surface。

| 路径                      | 主要职责                 | 备注                                                           |
| ------------------------- | ------------------------ | -------------------------------------------------------------- |
| `anemone-kernel/`         | 内核实现                 | tasking、内存管理、VFS、驱动、trap、syscall 等都应从这里开始找 |
| `anemone-abi/`            | 内核与用户态 ABI 边界    | ABI 变更通常应配套决策记录，至少也要有清晰日志                 |
| `anemone-rs/`             | Rust 用户态支持库        | 适合查看用户可见契约与 smoke 验证                              |
| `anemone-apps/`           | 用户态应用与复现程序     | 适合做定向验证和回归复现                                       |
| `anemone-apps/user-test/` | 测例脚本编排入口         | testsuits 章节中的脚本级测例集名称以这里的静态数组为准         |
| `conf/`                   | 平台、架构与 rootfs 配置 | 平台接线以这里为准                                             |
| `scripts/xtask/`          | 构建编排                 | 仓库构建应走 `just` 与 `xtask`，而不是临时 cargo 命令          |
| `build/`                  | 生成物与阶段性产物       | 可检查输出，但不要手改生成文件                                 |
| `docs/`                   | 当前有效的整理文档       | 包含当前事实页面、活动记录、testsuits 与公开草案规则           |

## 维护规则

只有在顶层所有权边界变化，或某个新的顶层区域已经成为日常开发的重要入口时，才更新这一页。
