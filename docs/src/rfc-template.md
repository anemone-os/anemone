# RFC 模板

复制本页内容时，替换所有占位字段与说明文字。

```md
# RFC-YYYYMMDD-short-slug

**Status:** Draft
**Owners:** name1, name2
**Last Updated:** YYYY-MM-DD
**Related:** 开发日志、问题条目、已有决策记录、相关 PR 或 issue

## Summary

用一段话说明这份 RFC 要解决什么问题，以及提议的核心方向。

## Context

说明当前背景、已有约束、已有实现状态，以及为什么这个问题现在必须被处理。

## Goals

- 本 RFC 明确要解决的目标 1。
- 本 RFC 明确要解决的目标 2。

## Non-Goals

- 本 RFC 明确不处理的内容 1。
- 本 RFC 明确不处理的内容 2。

## Proposal

按主题解释提议方案本身。必要时拆成多个小节，例如：

### Interfaces

描述对外接口、配置、ABI、行为边界或用户可见语义。

### Internal Design

描述内部结构、关键数据流、状态机、不变量或子系统边界。

### Migration / Rollout

描述如何从当前状态迁移到目标状态，以及中间阶段怎样保持系统可工作。

## Alternatives Considered

### Alternative A

说明这个方案是什么，以及为什么没有采用。

### Alternative B

说明这个方案是什么，以及为什么没有采用。

## Risks

- 风险 1，以及打算如何控制。
- 风险 2，以及打算如何控制。

## Open Questions

- 仍未决的问题 1。
- 仍未决的问题 2。

## Next Step

说明这份 RFC 下一步会进入什么状态，例如继续讨论、补原型、拆任务、转 ADR 等。
```# RFC 模板
