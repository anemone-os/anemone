# Testsuits

本章节记录用户态测例集在脚本层级上的通过情况与 bench 可运行性。

记录对象以 `anemone-apps/user-test/src/main.rs` 中的静态脚本数组为准，不展开脚本内部的具体子测例。

## 记录范围

- 功能性测例：记录是否能按预期运行并完成。
- 性能 Bench：记录是否可运行，以及关键结果摘要。

## 状态填写建议

功能性测例可使用：`未运行`、`进行中`、`部分通过`、`通过`、`失败`、`阻塞`。

性能 Bench 除了可运行性外，还应补充一列指标摘要，用于记录吞吐、时延或基准分数等关键结果。

## 维护规则

- 脚本数组变化时，同步更新本章节。
- 运行结果变化时，更新对应状态、日期与备注。
- 具体复现命令和环境差异，应记录在开发日志中，并在本章节按需引用。
