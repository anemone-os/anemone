## Developer Documentation

这个目录放的是公共仓库可见的开发文档，供开发者与 coding agent 共同使用。

核心规则很简单：

- `docs/` 只放当前有效的共享文档，以及确实需要公开协作的草案。
- 私有环境配置、个人草稿和未公开讨论不属于仓库文档的一部分。

`docs/src/` 下的 mdBook 只关注运行性知识：

- 当前哪些事实成立；
- 当前哪些东西坏了、缺了、或仍然脆弱；
- 一项修改应该如何验证；
- 一个重要决策为什么存在；
- 以及哪些公开草案仍处于讨论阶段。
- GitHub issues / PR 继续承担规划、讨论与任务编排；活动登记册只保留当前会改变诊断、验证或兼容性判断的活动事实。

它不会尝试维护一套与代码镜像重复的 API 文档。
